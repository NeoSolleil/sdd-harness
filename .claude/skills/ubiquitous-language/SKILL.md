# ubiquitous-language — ユビキタス言語（用語集）

仕様（discover/specify）・設計（plan）・実装（implement）・テストで**同じ言葉を同じ意味で使う**ための用語台帳。生きた辞書で、機能を発見するたびに育てる。

> **運用（使い方・規約・メンテナンス）は汎用。用語表はプロジェクト固有データ**なので、プロジェクトごとに埋める。

## このテンプレの埋め方（プロジェクト初期化時・AI向け指示）

1. `CLAUDE.md` の「このプロジェクトで何を作るか」を主入力に、**中心概念**から用語を起こす。
2. 下の「用語」表の**記入例（`（例）` 行）を削除**し、このプロジェクトの用語で埋める。
3. 各用語に **日本語・英語識別子・定義**を必ず与える（曖昧語・同義語を作らない）。
4. まだ概念が固まっていない段階では埋めきらなくてよいが、**用語未定義のまま仕様（specify）を書かない**。discover/specify で概念が出るたびに追記する。

## 使い方

- どのタスクでも**最初に参照**して用語を揃える。
- discover/specify で**新語が出たら、ここへ追加提案**する（定義＋英語識別子）。表記ゆれを作らない。
- 文書は日本語＋英語併記、**コードの識別子は英語**（下表の英語名に従う）。

## 用語（プロジェクト固有・要記入）

> 下は**書式を示す記入例**（`（例）` 行）。自プロジェクトの用語へ差し替える。

| 日本語 | 英語（識別子） | 定義 |
| --- | --- | --- |
| （例）注文 | order | （差し替え可）ユーザーが確定した購入の単位。金額と明細を持つ。 |

## 規約

- 1概念=1用語。同義語を増やさない（表記ゆれ・言い換えを1つに統一する）。
- **命名規則**: 各言語の標準的な命名慣習に従い、**語幹は言語間で共通**に保つ（同じ概念＝同じ語幹）。具体の大小文字（`snake_case` / `camelCase` / `PascalCase` 等）の使い分けは**使用言語による＝active プロファイルの領分**。例: 語幹 `order` → `order_date`（snake）／`orderDate`（camel）／`Order`（型・クラスは Pascal）。
- **振る舞いはビジネスアクションで命名する**: 状態の代入や UI 事象ではなく業務の意図を名前にする（❌ `setStatus(DONE)` → ⭕ `complete()`、❌ `handleClick` → ⭕ `submitOrder`）。コンポーネント・クラス名も見た目でなく役割で（❌ `RedButton` → ⭕ `PurchaseButton`）。
- これらは plan 段階で**ドメインのエンティティ／値オブジェクト**にほぼ対応する（中心概念はエンティティ、属性を表す概念は値オブジェクト候補）。
- DB テーブル名・カラム名・API のフィールド名も、原則この英語名（言語に応じた大小文字）に揃える。

## メンテナンス

新機能の discover で概念が増えたら、この表に行を足してから先へ進む（用語未定義のまま仕様を書かない）。
