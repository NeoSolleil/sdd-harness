# design — デザイントークン（Tailwind）

見た目を一貫させるための方針。**生の数値・色をハードコードせず、トークン（Tailwind ユーティリティ／`theme.extend`）で表現する**。**方法は汎用**で、具体的なトークン値はプロジェクト固有。

## 前提（導入）

- Tailwind 未導入のプロジェクトでは、まず `tailwindcss` を導入し `tailwind.config` を用意する。

## 運用原則（汎用）

- **役割で色を定義**（`primary` / `bg` / `surface` / `text` / `muted`、状態色 成功・失敗・警告 など）。個別 hex を画面に散らさない。
- スペーシングは Tailwind の 4px グリッド（`1`=4px, `2`=8px, `4`=16px…）に従う。任意 px（`mt-[13px]`）を使わない。
- タイポは少数のサイズに統一。数値表示は `tabular-nums` で桁揃え。
- 角丸・影は段階を絞って乱用しない。
- ダーク／ライト対応も**役割トークン経由**にし、個別色を散らさない。
- **不足したらトークンを足してから使う**（その場限りの値を入れない）。
- 命令的描画層（Canvas/WebGL 等）の色も、同じトークン値を参照して UI と揃える。

## プロジェクト固有のトークン（例・育てる）

| 役割 | 用途 |
| --- | --- |
| primary | 主要ボタン・強調 |
| bg / surface | 背景・パネル |
| text / muted | 本文・補助テキスト |
| 状態色（成功／失敗 など） | 結果のフィードバック表示 |

→ 具体値（hex・スケール）は UI 着手時に `tailwind.config` で確定し、ここに追記する。役割名は固定、値は差し替え可能に保つ。
